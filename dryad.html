Brief Description

Dryad is a modular application of distributed architecture. This engineering approach enables the components to provide high availability without cutting down on performance rates. It is composed of 9 modules whose communication is possible through both their :

  • Messaging APIs (AMQP messages containing a byteload of Google's Protocol Buffers through a RabbitMQ message bus) and their
  • Management APIs (web services controller allowing the user to configure the business flow as desired)
This kind of architecture renders the application language agnostic (the entire infrastructure can end up consisting of a polyglot environment) as far as the implementation of the components is concerned whilst simultaneously giving a steeper learning curve to the future developer and an oportunity for him/her to get involved more quickly. In order to eliminate any compatibility matrices we introduced Docker and containerization.

Dryad resides on both SQL(PostgreSQL) and noSQL(Elasticsearch) data bases for persistance. Due to the data galore Dryad produces we have integrated the Kafka message bus ,alongside a Zookeeper cordinator ,which we use as a data warehouse before the actual ETL procedures which will eventually store data in our reporting data bases.
